# CONTRIBUTING

## Welcome

Thank you for contributing to the AI-Powered Release Note Workflow Automation project.

This repository is the team's single source of truth. All important project information should be stored here rather than only in chats, emails, or personal files.

---

## Before You Start

1. Read the Project Charter.
2. Review the Team Roster.
3. Review open GitHub Issues.
4. Confirm your assigned workstream.

---

## Repository Structure

Please place content in the appropriate folder.

| Folder | Content |
|----------|----------|
| charter | Project charter |
| roster | Team roster |
| workflow | Workflow documentation |
| sample-data | Sample inputs and outputs |
| commands | Commands and prompts |
| skills | Skills documentation |
| agents | Agent workflow designs |
| mcp-integrations | MCP and integration concepts |
| validation | Validation and testing results |
| demo | Demo materials |
| meetings | Meeting notes |
| decisions | Project decisions |
| assumptions | Assumptions and constraints |

---

## How to Contribute

### Option A – Preferred

1. Create a branch.
2. Make your changes.
3. Submit a Pull Request.
4. Request review if needed.
5. Merge after approval.

Example branch names:

```text
feature/release-note-command
feature/reviewer-skill
feature/agent-workflow
```

### Option B – Workshop-Friendly

For documentation-only updates such as:

- Meeting notes
- Sample data
- Research findings
- Validation notes

Direct commits may be acceptable if agreed by the team.

---

## GitHub Issues

Use GitHub Issues to:

- Track work
- Record action items
- Raise questions
- Report blockers

Do not rely solely on chat messages for project tracking.

---

## Documentation Standards

When adding content:

- Use clear headings
- Include dates when relevant
- Document assumptions
- Document decisions
- Keep content concise and understandable

---

## Meeting Notes

Every meeting should produce:

- Decisions
- Action items
- Risks
- Next steps

Store notes in:

```text
meetings/
```

---

## Decision Logging

Important decisions should be documented in:

```text
decisions/
```

Do not keep critical decisions only in chat discussions.

---

## Guiding Principle

The goal is not to build the perfect release note generator.

The goal is to demonstrate how AI capabilities such as Commands, Skills, Agents, Marketplace tools, Plugins, and MCP can automate and improve a realistic documentation workflow.
